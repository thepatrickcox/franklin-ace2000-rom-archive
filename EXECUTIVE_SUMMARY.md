# Executive Summary

**Subject.** Four EPROMs retained since the mid-1980s by a former Franklin Computer Corporation QA tester, read and analyzed in June 2026.

**Identification.** The chips are a matched firmware set for the Franklin ACE 2000 series (October 1985), the Apple IIe-class machines Franklin built after losing *Apple v. Franklin* (3d Cir. 1983). The production chip is system ROM V5.0 ($E000-$FFFF). Beta chip P1 is V5.2, a later revision of the same ROM. Beta chip P2 is the companion system ROM ($C000-$DFFF). Beta chip P3 is floppy-disk controller firmware.

**The set interlocks.** The production ROM's reset sequence ends in `JMP $DEFE`, an address outside the chip. P2 contains, at $DEFE, the machine's cold-boot orchestrator, which initializes Apple IIe-class banking hardware, performs the warm-start validity check, and jumps back into the production ROM at $E000. The two images make 91 and 113 direct absolute calls into each other's address ranges. The boot path of the machine physically crosses from one preserved chip into another.

**Clean-room evidence.** In the 1983 litigation, Franklin's copied ROMs were identified partly by Apple copyright strings and the embedded word "Applesoft" surviving in Franklin's chips. These four chips contain no Apple attribution of any kind. The longest byte run shared with any of ~20 Apple II/II Plus reference ROMs is 56 bytes, and it is the Applesoft floating-point constant table — numeric coefficients, not program logic. No 64-byte window is shared anywhere. The firmware preserves Apple's published entry-point addresses (COUT $FDED, HOME $FC58, BELL $FF3A, and others) with different implementations behind them; Franklin's BELL is `LDA #$87 / JMP $FDED` against Apple's `LDA #$87 / JSR $FDED / RTS`. These ROMs are the physical record of what "Apple-compatible but independently written" meant in practice, two years after the courts required it.

**Forensic rigor.** Each chip was read multiple times; each 32 KB read contains the 8 KB payload twice, so every payload byte was reconstructed by majority vote over 6 to 14 samples. P2 and P3 read perfectly. The production chip and P1 show age-related weak cells concentrated in bits 7 and 0; six and seventeen byte positions respectively remain formally unresolved and are documented individually. Two semantic anchors prove P1's degradation mode (its hex-digit table reads `0123446789ABCDEF`; its product string reads `ACD` for `ACE`), while its V5.2 identity rests on evidence that degradation cannot produce: a bit-*set* version digit, a relocated entry point, and spatially clustered multi-byte code changes.

**A correction.** An earlier single read of the production chip yielded `JMP $5EFE`, an unmappable address. The byte is 0xDE in all ten current samples; the instruction is `JMP $DEFE`. The earlier value was a one-bit transient read error, in the exact bit position (bit 7) where this chip's weak cells cluster.

**Provenance.** The chips were retained personally by the contributor from his time at Franklin's Pennsauken, New Jersey facility, where he worked as a thirteen-year-old QA tester circa 1982-84. Chain of custody, label transcriptions, and photographs are documented in `PROVENANCE.md`.
